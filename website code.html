Deploy to GitHub Pages

Free hosting at haydenharms.github.io — takes about 5 minutes

01

Create a GitHub Account

Go to github.com and sign up. Choose a username — ideally haydenharms since it'll appear in your URL.

02

Create a Repository

Click the + icon → New repository. Name it exactly: haydenharms.github.io (replace with your username). Set it to Public.

03

Upload Your File

Inside the new repo, click Add file → Upload files. Drag your index.html file in and click Commit changes.

04

Enable GitHub Pages

Go to Settings → Pages. Under Source, select Deploy from a branch, choose main and / (root). Click Save.

05

Visit Your Site

After ~60 seconds, your site is live at haydenharms.github.io. To update it later, just re-upload the file.

06

Custom Domain (Optional)

Buy haydenharms.com on Namecheap (~$12/yr). In Pages settings, add it under Custom Domain. Takes 10 min to configure.

# Alternatively, if you prefer the command line:

git init
git add index.html
git commit -m "Initial site"
git branch -M main
git remote add origin https://github.com/haydenharms/haydenharms.github.io.git
git push -u origin main

# Then enable Pages in repo Settings → Pages
